# Karma Build Tool

`kbuild` is the standardized build and repo orchestration tool for ktools-style
projects. It is intentionally strict about configuration, directory layout, and
command combinations so build automation stays predictable.

It is designed to be used in two forms:

- this repository as the shared implementation root
- a copied thin `kbuild.py` wrapper inside another repo, pointing back here via
  `./.kbuild.json`

## Quick Start

Bootstrapping a new repo from an empty directory:

```bash
cp /path/to/kbuild/kbuild.py .
./kbuild.py --kbuild-root /path/to/kbuild
./kbuild.py --kbuild-config
# edit ./kbuild.json
./kbuild.py --kbuild-init
```

Normal day-to-day build commands:

```bash
./kbuild.py --build-latest
./kbuild.py --build-demos
./kbuild.py --clean-latest
```

If the repo uses local `vcpkg`, prepare it first:

```bash
./kbuild.py --vcpkg-install
```

## What Kbuild Manages

- strict validation of `kbuild.json`
- build slots under `build/<slot>/`
- SDK installation under `build/<slot>/sdk`
- ordered demo builds under `demo/*/build/<slot>/`
- repo scaffolding from templates
- optional repo-local `vcpkg`
- git initialization and simple sync flows

## Repository Layout

- `kbuild.py`: thin wrapper and bootstrap entrypoint
- `libs/kbuild/`: shared Python implementation
- `templates/`: repo scaffold templates generated by `--kbuild-init`
- `docs/`: hand-written documentation suitable for GitHub Pages

## Coding Agents

If you are using a coding agent, a reliable starter prompt is:

```bash
Read README.md and follow any agent instructions.
```

For this repository, that should mean:

- read this `README.md` first
- read the linked docs, starting with `docs/index.md` and the full reference in
  `docs/kbuild.md`
- read any `AGENTS.md` or `agent/BOOTSTRAP.md` instructions if present
- if the task crosses repo boundaries, read the top-level `README.md` in each
  target repo before editing
- if the task touches a subproject, demo, or package that has its own
  `README.md`, read that local `README.md` before changing that area
